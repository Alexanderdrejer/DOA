## Question 1
### Consider the following algorithms: 

```cpp
bool alg1(const vector<int>& arr) {
    int count = 0;
    for (size_t i = 0; i < arr.size(); i++) {
        for (size_t j = i + 1; j < arr.size(); j++) {
            if (arr[i] == arr[j]) {
                count++;
            }
        }
    }
    return count > 0;
}
```

```cpp
vector<int> alg2(const vector<int>& arr) {
    vector<int> prefixSums;
    int total = 0;
    for (int num : arr) {
        total += num;
        prefixSums.push_back(total);
    }
    return prefixSums;
}
```

```cpp
void alg3(vector<int>& arr) {
    size_t n = arr.size();
    for (size_t i = 0; i < n; i++) {
        bool swapped = false;
        for (size_t j = 0; j < n - i - 1; j++) {
            if (arr[j] > arr[j + 1]) {
                swap(arr[j], arr[j + 1]);
                swapped = true;
            }
        }
        if (!swapped) break;
    }
}
```

## Provide an answer to each the following, for each of the above algorithms:
### 1. Explain what the goal of the algorithm is (e.g., does it find the minimum element in the input vector?).

Den første algortime finder dupletter. Den sikrer en søgning efter dupletter, ved at bruge j + 1 som betingelse, da den ellers ville sammenligne de præcis sammen indekser og dermed de samme tal.

Den anden algoritme kigger igennem en vektor og summerer alle tal til en samlet liste. Antag der er 10, 20, 30 på den vektor der lægges ind: 0+10=10. 10 + 20 = 30, 30+30 = 60.

Den tredje algortime repræsenterer bubblesort. Den løber vektoren der er givet som argument igennem vha .size() repræsenteret af n. I det nestede for-loop er betingelsen, at hvis j er mindre end størrelsen på vektoren - i - n - 1. n er stadig bare størrelsen af min vektor, mens -1 skal sikre at algoritmen ikke går "out of bounds".
